## Code Changes

### `src/laser_trim_analyzer/gui/widgets/chart.py`
- Added `_sync_figure_to_widget()` and call it from `clear()` so every plot
  is rendered at the actual widget pixel size instead of the figure's
  initial figsize. This fixes the sigma scatter "top-left ghost" symptom.
- Resize handler now refuses to clobber the size when the widget is still
  1x1 during initial layout (falls back to the parent's size).

### `src/laser_trim_analyzer/gui/pages/trends.py`
- `_show_drift_timeline()` (already done earlier this session): preserve
  facecolor across clears, color-code markers, sort by most recent drift,
  dynamic figure height.
- New public method `show_drift_tab()` so the dashboard's Drift Alerts
  card can navigate straight into the Drift view.

### `src/laser_trim_analyzer/gui/pages/dashboard.py`
- Drift Alerts card is now clickable (`cursor="hand2"`, arrow in title,
  `<Button-1>` binding). Clicking jumps to Trends > Drift.

### `src/laser_trim_analyzer/gui/pages/specs.py`
- Added `on_hide()` that destroys the 300+ rendered row frames so they
  don't stay alive in the geometry manager and slow down the rest of the
  session.

### `src/laser_trim_analyzer/gui/pages/smoothness.py`
- When a record has no track data, the chart now shows a clear placeholder
  instructing the user to re-process the source file (or run the repair
  script) instead of rendering a blank panel.

### `src/laser_trim_analyzer/ml/manager.py`
- Replaced the per-track `UPDATE ... WHERE id = ?` loop in
  `apply_to_database()` with a single `bulk_update_mappings()` call. With
  ~80K tracks across ~200 models this was issuing ~80K individual
  statements inside one transaction, hence the >1 hour runtime. Now one
  bulk UPDATE per model.

### `src/laser_trim_analyzer/database/manager.py`
- Added `get_smoothness_files_missing_tracks()` mirroring the existing
  Final Test helper.
- Added `update_smoothness_tracks(smoothness_id, tracks)` so the repair
  script can backfill the per-position arrays.
- Added `CREATE INDEX IF NOT EXISTS idx_ft_file_date` to the startup
  migration so the Final Test page's `ORDER BY file_date DESC LIMIT 500`
  query stops full-scanning + sorting.

### `src/laser_trim_analyzer/database/models.py`
- Declared `Index('idx_ft_file_date', 'file_date')` on `FinalTestResult`
  so future databases get the index from `create_all()` too.

### `scripts/fix_smoothness_tracks.py` (new)
- Standalone script: finds smoothness records with 0 tracks, re-parses
  the original .xlsx via `SmoothnessParser.parse_file`, and writes the
  per-track data back via `update_smoothness_tracks`.

## How to Use the Smoothness Repair Script

```
python scripts/fix_smoothness_tracks.py
```

It will:
1. Query the DB for any `SmoothnessResult` with 0 rows in `smoothness_tracks`.
2. For each, re-parse the original .xlsx file at the recorded `file_path`.
3. Write the tracks back via `update_smoothness_tracks`.

If a source file no longer exists at the recorded path, the record is
skipped and reported at the end - re-process those via the Process Files
page once you have the source files available.
