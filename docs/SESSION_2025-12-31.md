# Session Notes - 2025-12-31

## Summary
Bug fixes and UI improvements for chart resizing, Compare page spec limits, and Trends page anomaly handling.

## Completed

### 1. Fixed macOS Tcl/Tk Path Issue
- App wasn't starting due to Python 3.14 (uv) not finding Tcl/Tk libraries
- Added automatic Tcl/Tk path detection in `__main__.py` for macOS
- Sets `TCL_LIBRARY` and `TK_LIBRARY` environment variables before tkinter imports

### 2. Fixed Analyze Page Chart Resizing
- Changed from `CTkScrollableFrame` to regular `CTkFrame` for proper resizing
- Removed `minsize` constraints that prevented shrinking
- Chart area now resizes fully with the window

### 3. Added Spec Limits to Compare Page Charts
- Compare page charts now show red dashed spec limit lines (like Analyze page)
- Gets spec limits from Final Test track first, falls back to Trim track
- Added `optimal_offset` to Trim track data in database query
- Fill between spec limits for visual clarity

### 4. Excluded Anomalies from Trends Page Charts
- Anomalies (trim failures with linear slope) now excluded from scatter chart
- Added "Anomalies" count to stats bar (purple when > 0)
- Rolling average calculated from normal samples only
- Shows placeholder message if all samples are anomalies

## Known Issues

- **Root Cause**: 9 linked Final Test records point to old Trim records (IDs 7, 36, 59, 70, 73, 76) that have no track data stored
- Track data storage was added later; these old records predate it
- **Solution**: Re-process the original Trim files to update records with track data
- 775/807 Final Tests have track data; the 9 linked ones are the exceptions

### Models with Missing Track Data
- 7054, 8213-1, 8551, 8655: All Trim records have 0 tracks (need re-processing)
- 8652, 8753: Have some Trims with track data, but linking picked old ones
